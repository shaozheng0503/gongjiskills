## 一、TL;DR 一分钟速览

**一条命令跑起来一个 LLM 推理服务：**

```bash
gongji deploy --template qwen3.5-9b -n my-llm --ttl 3600 --json
# → {"task_id": 1926525, "status": "Running",
#    "urls": [{"url": "https://deployment-...550w.link", "port": 8000}],
#    "ttl_seconds": 3600}
```

发生了什么：

```
你的指令
   │
   ▼
┌──────────────────────────────────────────────────┐
│  gongji CLI（本项目）                              │
│  1. 查询所有区域的 4090 → 选最便宜且有库存的        │
│  2. RSA-SHA256 签名，调共绩 Open API 创建任务        │
│  3. 轮询任务状态，实时打印拉镜像/启容器事件          │
│  4. 任务就绪，解析访问 URL，返回 JSON               │
│  5. 后台守护进程，1 小时后自动 stop（--ttl）         │
└──────────────────────────────────────────────────┘
   │
   ▼
公网可访问的推理服务 → Agent / 代码直接调用
```

**无需**：看 API 文档、手写 RSA 签名、自己挑 GPU、轮询状态、写错误处理、写重试、写释放逻辑。

## 二、背景：共绩算力是什么

[共绩算力](https://www.gongjiyun.com)（原"共绩云"）是一个**按秒计费**的 GPU 弹性算力平台，核心卖点：

- **资源丰富**：4090 / 5090 / H800 / H20 / A800 等主流卡，多个区域（广东/河南/河北/安徽/四川等）
- **按秒计费**：部署启动后按秒扣费，释放即停止扣费，不存在包年包月或最低消费
- **预制镜像库**：平台镜像仓库 `harbor.suanleme.cn` 预缓存了数十个常用 AI 镜像（vLLM、ComfyUI、FLUX、Qwen、Whisper、MinerU 等），拉取速度远快于从 Docker Hub
- **公网访问**：部署后自动生成形如 `https://deployment-xxx.550w.link` 的 HTTPS 地址，无需额外配置

平台有两种用法：
1. **控制台**（浏览器手动操作）— 适合临时、偶尔用
2. **Open API**（程序调用）— 适合自动化、批量、Agent 调用

本项目封装的是 **Open API** 路径，目标是把「从看 API 文档到真能跑起来」之间的所有脏活累活都抹平。

## 三、它解决什么问题

### 3.1 日常场景

这些场景，共绩都能用 GPU 弹性部署解决，但每次都要走一遍控制台很烦：

| 场景 | 需求 | 为什么需要弹性 |
|------|------|----------------|
| 跑一次 Qwen 70B 推理验证 | 一两个小时 | 本地显存不够，包月不值 |
| 批量生成 500 张 SD 图 | 半天 | 有任务就开，没任务就关 |
| 给客户演示一个 ComfyUI 工作流 | 1 小时 | 演示完立刻释放 |
| 月末批量处理一批 PDF OCR | 1 天/月 | 非高频，不值得持有 GPU |
| CI 里跑一次模型评估 | 30 分钟 | 只有 PR 时才需要 |
| Agent 接到任务需要调用视觉模型 | 按需 | Agent 自己决定什么时候要、要多久 |
| 临时给团队开个 Whisper 转录服务 | 一下午 | 用完就拆 |

### 3.2 手动操作的痛点

传统的「打开控制台」流程：

```
① 登录共绩控制台
② 浏览 GPU 列表，对比价格和库存
③ 挑一款 → 新建部署
④ 选镜像（记得镜像地址吗？）
⑤ 填端口、环境变量、启动命令
⑥ 等 2-5 分钟看是否拉镜像成功
⑦ 拉失败 → 回去看事件日志 → 调整参数 → 重新部署
⑧ 部署好了 → 复制访问 URL → 回到代码里
⑨ 用完 → 回到控制台 → 停止部署（经常忘！）
```

最后一步**最容易忘**。一个忘记释放的 4090 部署，**一晚上烧掉 30 元**，一个月几百元。

### 3.3 这个 Skill 的回答

把上面 9 步压缩成：

```bash
gongji deploy --template qwen-image -n my-img --ttl 3600 --json
# 一小时后自动释放，永远不会因为忘记 stop 烧钱
```

如果你是 Agent 开发者，这意味着：**你的 Agent 可以自己决定什么时候要 GPU、要多久、用完自己释放**，无需人工介入。

## 五、为什么对 AI Agent 友好

Agent（Claude / GPT / 自研 LLM Agent）调用外部工具最怕三件事：

1. **输出没法解析** — 人类友好的文本，Agent 看着一头雾水
2. **错误没人告诉怎么修** — 返回个"error code 10003"，Agent 只能放弃
3. **命令参数要背** — 文档五页，每次调用都要看

这个 Skill 针对性解决：

### 5.1 `--json` 统一契约

所有命令都支持 `--json`。成功和失败**都返回合法 JSON**，格式稳定：

```bash
# deploy 成功
$ gongji deploy --template vllm -n svc --json
{
  "task_id": 1926525,
  "status": "Running",
  "urls": [{"url": "https://deployment-xxx.550w.link", "port": 8000}],
  "ttl_seconds": 3600
}

# deploy 失败
$ gongji deploy --template not-exist -n svc --json
{"error": "模板 [not-exist] 不存在，用 gongji images 查看可用模板"}

# list
$ gongji list --json
[{"task_id": 1926525, "task_name": "svc", "status": "Running", "urls": [...]}]

# stop
$ gongji stop 1926525 -f --json
{"task_id": 1926525, "action": "删除", "ok": true}

# 批量释放
$ gongji stop --all -f --json
{"stopped": [1, 2, 3], "failed": []}
```

Agent 的调用代码简化到：

```python
import json, subprocess
result = subprocess.run(
    ["gongji", "deploy", "--template", "vllm", "-n", "svc", "--json"],
    capture_output=True, text=True
)
data = json.loads(result.stdout)
if "error" in data:
    handle_error(data["error"])
else:
    use_url(data["urls"][0]["url"])
```

### 5.2 模板库：不用背镜像地址

47 个预制模板按 11 个分类组织，Agent 先查分类、再选模板：

```bash
# 第一步：看分类
gongji images categories --json
# → {"llm": 10, "image-gen": 10, "video": 3, ...}

# 第二步：按用户需求挑分类
gongji images --category llm --json
# → 返回 10 个 LLM 模板的完整信息（镜像、端口、推荐 GPU、启动命令）

# 第三步：挑一个部署
gongji deploy --template qwen3.5-9b -n my-llm --json
```

每个模板**都带默认的启动命令和端口**，Agent 不用再去猜「vLLM 的启动命令是什么」、「端口开几个」。

### 5.3 错误带"怎么修"

底层 API 返回 `"code": "10003", "message": "signature invalid"`，这个 Skill 会翻译成：

```
签名验证失败：公钥可能未上传到控制台
  → 检查 ~/.gongji/public.pem 是否与平台一致
  → 重新上传公钥后运行 gongji init --force
```

支持的错误映射：

| API 错误 | 翻译后建议 |
|----------|-----------|
| Token 失效 / expired | "请登录控制台重新生成 API 密钥，再运行 `gongji init --force`" |
| 签名错误 | "请检查公钥是否已上传到控制台" |
| 库存不足 / inventory | "当前区域无库存，尝试换 `-r 广东` 或 `-g 3090`" |
| 余额不足 / insufficient | "账户余额不足，请前往控制台充值" |
| 资源不存在 / not found | "资源不存在，用 `gongji list` 确认 task_id" |
| 网络超时 / 5xx | **自动重试 2 次（指数退避）**，不需要 Agent 自己处理 |

### 5.4 TTL 防 Agent 烧钱

Agent 最大的风险是：它起了个任务，然后任务链路崩了 / 会话结束 / 忘记 stop，GPU 一直跑，月底账单爆炸。

`--ttl 3600` 会**启动独立后台守护进程**：

```bash
gongji deploy --template vllm -n tmp --ttl 3600 --json
# → {"task_id": ..., "ttl_seconds": 3600, "auto_release_pid": 12345}
```

- 独立进程，CLI 退出不影响
- 到期自动调 stop_task
- 日志写到 `~/.gongji/ttl/<task_id>.log`
- 可手动取消：`kill <auto_release_pid>`

## 七、典型使用场景案例

### 场景 1：AI Agent 按需开 LLM 推理

**需求**：Claude Code Agent 要调用 Qwen3.5-9B 跑一个批量分类任务。

```bash
# Agent 自己调用
gongji deploy --template qwen3.5-9b -n batch-classify --ttl 1800 --json
# → {"task_id": 1926525, "status": "Running",
#    "urls": [{"url": "https://dep-xxx.550w.link", "port": 8000}]}
```

然后 Agent 用拿到的 URL：

```python
import openai
client = openai.OpenAI(
    base_url="https://dep-xxx.550w.link/v1",
    api_key="none"  # vLLM 不校验
)
for item in batch:
    resp = client.chat.completions.create(
        model="Qwen/Qwen3.5-9B",
        messages=[{"role": "user", "content": f"分类：{item}"}]
    )
# 30 分钟后自动释放，成本约 ¥0.5
```

### 场景 2：一次性生成 1000 张 SD 图

```bash
gongji deploy --template sd3.5-comfyui -n bulk-gen -c 1 --ttl 7200 --json
# → URL: https://dep-xxx.550w.link/v1/workflows/run
```

脚本批量发请求，2 小时内跑完，自动释放。成本 ¥2-3。

# 轮询就绪
import time
while True:
    detail = client.task_detail(task_id)
    if detail["data"]["status"] == "Running":
        url = detail["data"]["services"][0]["remote_ports"][0]["url"]
        break
    time.sleep(5)

# ... 用 url ...

client.stop_task(task_id)

## 九、和直接调 Open API 的差别

如果你问"为啥不直接拿 [Open API 文档](https://www.gongjiyun.com/docs/platform/openapi/zx3iwhbv1i8sxdkeiapcprxhn8d/) 自己写？"

当然可以，但你要自己处理以下所有事情：

| 需要自己写 | Skill 帮你做了 | 对应文件 |
|-----------|----------------|---------|
| RSA-SHA256 签名（拼接 `path\nversion\ntimestamp\ntoken\ndata`） | 封装好的 `build_headers()` | `auth.py` |
| 管理私钥 / 公钥 / token 的生成和存储 | `gongji init` 一步到位 | `cli.py` |
| 权限加固（600/700） | 自动检测 + 自动设置 | `auth.py` |
| 调 `/api/deployment/resource/search` 查 GPU | 封装为 `search_resources()` | `client.py` |
| 遍历返回结果按价格排序 | `_find_cheapest()` | `cli.py` |
| 调 `/api/deployment/task/create` 创建任务 | `create_task()` | `client.py` |
| 轮询任务状态直到 Running | 内置轮询 + 超时处理 | `cli.py` |
| 解析 `remote_ports` 提取 URL | `_get_urls()` | `cli.py` |
| 解析事件（镜像拉取失败、OOM）| `pod_event()` + 友好展示 | `client.py` + `cli.py` |
| 错误码 `0000` / `10001` / ... 翻译 | `_friendly_error()` 映射表 | `client.py` |
| 瞬时错误 / 5xx / 超时重试 | 指数退避 ×2 | `client.py` |
| 到时间自动释放任务 | `--ttl` 独立守护进程 | `cli.py` |
| 输出规范化、JSON 契约 | `--json` 统一 | `cli.py` |
| 配 47 个预制镜像的启动参数 | 内置 `templates.py` | `templates.py` |
| 分类组织、检索 | `group_by_category()` + `--category` | `templates.py` + `cli.py` |

**粗算**：自己从 0 写到"能用"需要 2-3 天，写到"Agent 能稳定用"需要 1-2 周。用这个 Skill 是 0 成本。

## 十、成本控制机制

### 10.1 按秒计费的现实

共绩部署**一启动就按秒扣费**。举例（实际以平台价格为准）：

| GPU | 折扣价 | 1 小时 | 1 天（忘了关）| 1 周 |
|-----|-------|--------|--------------|------|
| 4090 单卡 | ~¥1 | ¥1 | ¥24 | ¥168 |
| 4090 x4 | ~¥4 | ¥4 | ¥96 | ¥672 |
| H20 x8 | ~¥30+ | ¥30+ | ¥700+ | ¥5000+ |

Agent 或人类忘记释放一个 4×4090 任务一晚上，就可以烧掉几百块。

### 10.2 三道防护

**防线 1：`--ttl` 强制到期释放**

```bash
gongji deploy --template qwen3.5-9b -n tmp --ttl 3600 --json
```

后台独立进程到时间自动 stop，CLI 退出也不影响。

**防线 2：`gongji status <id>` 实时看花了多少**

```
$ gongji status 1926525
任务ID:   1926525
状态:     Running
GPU:      4090 x1 (显存 24.0G)
已花费:   36.44 元
访问地址: https://deployment-xxx.550w.link
```

**防线 3：`gongji stop --all` 一键全关**

定时跑这个命令（比如每天下班前），或者 Agent 在会话结束时调用：

```bash
gongji stop --all -f --json
# → {"stopped": [1, 2, 3], "failed": []}
```

### 10.3 推荐实践

- **Agent 调用**：**永远带 `--ttl`**，默认 1 小时（`--ttl 3600`），长任务视情况调
- **人类操作**：加到 shell alias，比如 `alias gongji-stop-all="gongji stop --all -f"`
- **CI 场景**：把 `gongji stop` 放在 `finally` / `trap` 里，保证一定执行
- **定时任务**：用 cron 每天早上检查未释放任务（`gongji list --json`），报警或自动清理

## 十一、安全设计

### 11.1 凭据保护

| 项目 | 保护措施 |
|------|---------|
| `~/.gongji/` 目录 | 权限 `700`（仅当前用户可访问） |
| `private.key` 私钥 | 权限 `600`（仅当前用户可读写） |
| `config.json`（含 token） | 权限 `600` |
| `public.pem` 公钥 | 权限 `644`（可读，不敏感） |
| 启动时权限检测 | 加载配置时自动检查，过宽输出警告 + 修复命令 |

### 11.2 Token 传入方式

**推荐顺序**（安全性从高到低）：

1. **环境变量**（最推荐）：`GONGJI_TOKEN=xxx gongji init --force`
   - 不会进 shell history
   - 不会写入文件（只用来 init）
2. **stdin 管道**：`echo $TOKEN | gongji init --force`
3. **命令行参数**：`gongji init --token xxx`
   - 会进 history，风险较高
4. **交互输入**（默认）：不传 token 会提示输入
   - 终端会打星号

### 11.3 请求签名

每次请求都用 RSA-SHA256（PKCS1v15）签名，签名内容包括：

```
path\nversion\ntimestamp\ntoken\ndata
```

`timestamp` 由客户端本地时间生成，每次不同。即便截获请求，也不能重放（时间戳验证 + 数据完整性）。

## 十二、可靠性保证

### 12.1 自动重试

瞬时错误自动重试 2 次，指数退避（1s → 2s）：

- 连接失败（`ConnectionError`）
- 请求超时（`Timeout`）
- HTTP 5xx（500/502/503/504）

Agent 不需要自己写重试逻辑。

### 12.2 部署等待机制

`deploy` 默认会等待任务进入 `Running` 状态（最多 5 分钟）：

- 每 5 秒轮询一次状态
- 每次查询如果失败，重试，连续 3 次失败才报错（防止网络抖动）
- 任务进入 `End` / `Other` 异常状态时，**自动拉取失败原因**（Pod 事件）并显示

### 12.3 守护进程独立性

`--ttl` 的守护进程是**独立子进程**（`start_new_session=True`）：

- CLI 退出不影响守护进程
- 独立 PID 写在 `~/.gongji/ttl/<task_id>.pid`
- 日志写在 `~/.gongji/ttl/<task_id>.log`
- 可以用 `kill <pid>` 手动取消

### 12.4 配置容错

- 私钥文件不存在 → 友好报错，引导 `gongji init`
- token 格式无效 → 引导重新生成
- 系统时间不准 → 签名失败时会提示（时间戳误差大于 5 分钟 API 会拒绝）

## 十四、适合 / 不适合的场景

### 14.1 适合

- ✅ **AI Agent 按需开 GPU**：Claude / GPT / 自研 Agent 自动调用
- ✅ **开发期验证模型**：临时跑跑开源模型试试效果
- ✅ **一次性批量作业**：跑一批推理/生成/解析，做完就散
- ✅ **CI / 自动化脚本**：PR 触发的评估、定期跑的数据处理
- ✅ **演示 / POC**：临时给客户演示，演完释放
- ✅ **突发业务**：临时压力 / 活动高峰
- ✅ **团队共享 GPU**：按需开放 API，用完释放，避免独占

### 14.2 不适合

- ❌ **长期稳定的生产服务**：持续运行的 API，建议直接买固定实例或专属容器，单价更便宜
- ❌ **强 SLA 的关键业务**：弹性资源库存会波动，某些区域/型号可能临时无货
- ❌ **数据库 / 有状态服务**：弹性部署不保证数据持久化（需自行挂载共享存储）
- ❌ **需要极端低延迟的实时交互**：每次冷启动要 30-60 秒拉镜像

## 十五、常见问题

**Q: 我没有共绩账号，可以白嫖吗？**
A: 不能。必须注册并充值（最低 ¥5-10），但按秒计费，试用成本极低。

**Q: 部署失败，报镜像拉取失败怎么办？**
A: `gongji logs <id> --events` 查事件。常见原因：
  - 用了 docker hub / ghcr 的上游镜像，平台内网拉不到 → 换用 `gongji images` 里的预制模板
  - 镜像地址拼错 → 仔细检查
  - 私有镜像没授权 → 用 `repository_username` / `repository_password`

**Q: Agent 调用时怎么知道任务跑起来了？**
A: `gongji deploy --json` 会**等到 Running** 才返回（默认等 5 分钟）。如果想立刻返回 task_id，加 `--no-wait`。

**Q: 能部署多节点（多副本）吗？**
A: 能。`gongji deploy --template xxx -n svc --points 3` 开 3 副本，平台会做负载均衡（但多数预制镜像标注"单节点"的不支持）。

**Q: 我的自定义启动命令怎么传？**
A:
```bash
gongji deploy <image> -n svc \
  --start-cmd "/bin/bash" \
  --start-args "-c 'vllm serve my-model --port 8000'"
```

**Q: 怎么挂载持久化存储？**
A: 目前 CLI 不支持。要挂 `storage_config` 请走 Python API `client.create_task(storage_config=[...])`。

**Q: 我公司只能访问内网，能装吗？**
A: 能，但需要从内网能访问 `openapi.suanli.cn` 和共绩镜像仓库。先做网络联通性测试。

**Q: `--ttl` 守护进程会在什么情况下失效？**
A: 机器关机 / 账户注销 / 被 `kill` 时会丢。建议重要任务用 cron 做二次保险。

**Q: 怎么切换多个账号？**
A: 目前 `~/.gongji/config.json` 是单账号。想切换用 `GONGJI_DIR=~/.gongji-acc2 gongji init` 分离配置（需要改 auth.py 支持，目前是硬编码 `~/.gongji`）。欢迎 PR。
